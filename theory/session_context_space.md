Рассмотрим **поток данных и управления** через сессию Linux — от ввода пользователя до вывода результата, проходя через все уровни (ядро, память, процессы).

Ниже — **подробная диаграмма процесса** в терминах "вход → обработка → выход" для типичной сессии (например, вы запустили `ls -l` в терминале).

## Таблица: "Что на входе, что на выходе" для разных уровней

| Уровень | Вход | Обработка | Выход |
|---------|------|-----------|-------|
| **Аппаратура** | Скан-коды клавиш | Прерывание CPU | Символ в буфер драйвера |
| **Драйвер TTY** | Сырые байты | Сохранение в кольцевой буфер | Структура `struct input` |
| **Дисциплина линии** | Сырые байты + спецсимволы | Обработка Ctrl+C (SIGINT), редактирование строки | Каноническая строка (`"ls -l\n"`) |
| **Ядро (TTY/PTY)** | Строка команды | Поиск управляющего процесса (SID) | Сигнал SIGTTIN (если фон) или данные процессу |
| **Сессия/Группы** | Команда | Определение, какой процесс在前台 | Переключение контекста на процесс |
| **Память процесса** | Аргументы (`argv[]`) | Загрузка в стек | Системный вызов `execve()` |
| **Системные вызовы** | `read()`, `write()` | Переход в ядро (syscall) | Копирование данных между user/kernel space |
| **Дисплей/Сеть** | Буферизованный вывод | Рендеринг/Кодирование | Кадры на монитор / пакеты в сокет |

## Ключевые структуры памяти (для углубления)

В ядре Linux сессия представлена следующими структурами (упрощённо):

```c
// В ядре: include/linux/sched.h
struct task_struct {
    pid_t pid;                // PID процесса
    pid_t tgid;               // Thread group ID
    struct pid_link pids[PIDTYPE_MAX];
    struct task_struct *group_leader; // Лидер группы
    
    // СВЯЗЬ С СЕССИЕЙ:
    unsigned int sessionid;   // SID (Session ID)
    struct tty_struct *tty;   // Управляющий терминал
};

// Сессия (группа сессий)
struct session_struct {
    pid_t sid;                // Идентификатор сессии
    struct tty_struct *tty;   // Управляющий терминал
    struct pid *pids;         // Список процессов в сессии
    atomic_t refcount;        // Счётчик ссылок
};
```

**Как это работает в памяти:**
1. Когда вы нажимаете клавишу → прерывание → драйвер клавиатуры → ядро помещает скан-код в буфер **input_handler**.
2. TTY-драйвер читает из буфера → преобразует в символ → сохраняет в **line discipline buffer**.
3. При нажатии Enter → ядро проверяет **struct tty_struct** → находит **foreground process group** (PGID) → отправляет данные процессу.
4. Процесс `ls` в userspace получает данные через `read(0, ...)` → выполняет системный вызов `getdents()` (чтение директории) → формирует вывод.
5. `ls` вызывает `write(1, ...)` → ядро копирует данные из **user space** (heap/stack процесса) в **kernel space** → TTY-драйвер → терминальный эмулятор → X11/Wayland → GPU → монитор.

## Практический пример с командами

Выполните эти команды, чтобы увидеть "вход и выход" в действии:

```bash
# 1. Посмотреть текущую сессию и группы процессов
ps -eo pid,sid,pgid,tty,comm | head -5
# Выход: покажет SID (Session ID) и PGID для каждого процесса

# 2. Отследить системные вызовы при запуске ls
strace -f -e trace=read,write,execve ls -l
# Вход: strace перехватывает все системные вызовы
# Выход: увидите, как ls читает директорию (getdents) и пишет в STDOUT

# 3. Посмотреть, как сигналы влияют на сессию
tty  # показывает текущий терминал, например /dev/pts/2
echo "hello" > /dev/pts/2  # прямой вывод в терминал другой сессии

# 4. Исследовать память процесса
cat /proc/self/maps  # показывает виртуальное адресное пространство текущего процесса
```

## Итог: полный цикл "вход → выход" сессии

**Вход (Input Flow):**
```
Физическая клавиша → Прерывание CPU → Драйвер клавиатуры → 
Буфер ввода TTY → Дисциплина линии (обработка Ctrl+C/Backspace) → 
Канонический буфер → Поиск foreground процесса → 
Копирование данных в память процесса (user space)
```

**Обработка (Processing):**
```
Процесс читает данные (read syscall) → Парсит команду → 
Выполняет логику (системные вызовы: open, readdir, stat) → 
Формирует вывод в буфере процесса
```

**Выход (Output Flow):**
```
Процесс вызывает write syscall → Копирование из user space в kernel space → 
Буфер TTY → Дисциплина линии (добавление \r) → 
Драйвер PTY/TTY → Терминальный эмулятор → 
GUI (X11/Wayland) → Видеодрайвер → Монитор
```

Это полный **контекст памяти сессии** от ввода до вывода через все уровни Linux.
